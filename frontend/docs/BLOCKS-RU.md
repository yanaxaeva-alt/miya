# Памятка по блокам Miya Editor

Эта памятка переводит названия блоков палитры на русский и объясняет, зачем они нужны. В интерфейсе оставлены короткие технические названия, потому что они совпадают с MiaOS JSON и backend-событиями.

## Input / Output

| Блок | По-русски | Что означает |
|------|-----------|--------------|
| `START` | Старт / вход | Начальная точка графа. Сюда попадает текст из Run Console. |
| `END` | Финиш / выход | Конечная точка графа. После неё run считается завершённым. |

## Agents

| Блок | По-русски | Что делает |
|------|-----------|------------|
| `Планировщик` | Планировщик | Разбивает задачу на шаги и решает, что делать дальше. |
| `Исполнитель` | Исполнитель | Выполняет основную работу и формирует ответ/результат. |
| `Критик` | Проверяющий / критик | Проверяет результат на качество, безопасность и ошибки. В MiaOS JSON это `type: critic`. |
| `Память` | Память | Сжимает или извлекает важные факты для последующих шагов. |
| `Восприятие` | Восприятие | Разбирает вход пользователя: намерение, сущности, контекст. |

## Safety

| Блок | По-русски | Что делает |
|------|-----------|------------|
| `Согласование` | Ручное согласование | Останавливает граф перед рискованным действием. Создаёт запрос в Approval Queue. |

Типичные `action_class`:

- `publish` — публикация или внешняя отправка.
- `delete` — удаление.
- `finance` — финансовое действие.
- `self_modify` — изменение кода/системы/guardrails.

## Tools

| Блок | По-русски | Backend tool | Что делает |
|------|-----------|--------------|------------|
| `Web search` | Веб-поиск | `web_search_mock` | Mock-поиск. Без настоящего интернета, нужен для безопасной отладки tool flow. |
| `Draft` | Черновик | `create_draft` | Создаёт черновой текст из входа. |
| `Read file` | Чтение файла | `read_file_sandbox` | Читает файл только внутри sandbox. |
| `Write file` | Запись файла | `write_file_sandbox` | Пишет файл только внутри sandbox. |

## Шаблоны

| Шаблон | По-русски | Когда использовать |
|--------|-----------|--------------------|
| `mia-minimal` | Минимальный граф Mia | Базовая проверка: `START → Планировщик → Исполнитель → Согласование → END`. |
| `draft-with-tools` | Черновик с инструментами | Проверка tool nodes: `Web search → Draft → Согласование`. |
| `chat-memory-loop` | Чат с памятью | Эксперименты с восприятием, памятью и ответом persona. |

## Важное правило

Названия на карточках можно менять для удобства, но технические поля лучше не трогать без необходимости:

- `tool_name` должен совпадать с backend registry.
- `action_class` должен совпадать с safety policy.
- `role` влияет на prompt и тип экспортируемого узла.
- `START` и `END` остаются техническими id входа и выхода в MiaOS JSON.
